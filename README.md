# 区块链试卷保密系统 v1.0

基于区块链技术的学校试卷保密审批管理系统，融合多种密码学技术，为高校提供安全可靠的试卷全生命周期管理解决方案。

---

## 区块链安全特性

### 为什么选择区块链？

传统试卷保密系统面临的最大风险是**中心化存储**带来的单点失效和数据篡改问题。本系统采用区块链架构，从根本上解决这些安全隐患：

```

├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│    │  Node 1  │◄────►│  Node 2  │◄────►│  Node 3  │           │
│    │ (教务处)  │      │ (系办公室) │      │ (院办公室) │           │
│    └────┬─────┘      └────┬─────┘      └────┬─────┘           │
│         │                 │                 │                  │
│         ▼                 ▼                 ▼                  │
│    ┌──────────┐      ┌──────────┐      ┌──────────┐           │
│    │  MySQL   │      │  MySQL   │      │  MySQL   │           │
│    │ (加密存储) │      │ (加密存储) │      │ (加密存储) │           │
│    └──────────┘      └──────────┘      └──────────┘           │

│              └──────────────┘                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 区块链带来的六大安全保障

| 安全特性 | 技术实现 | 安全价值 |
|---------|---------|---------|
| **去中心化存储** | 多节点数据同步 | 单节点被攻破不影响整体安全 |
| **数据不可篡改** | 哈希链+数字签名 | 任何修改都会被检测到 |
| **操作可追溯** | 区块链审计日志 | 完整记录谁、何时、做了什么 |
| **共识机制** | 多节点验证 | 防止单方面伪造数据 |
| **抗抵赖性** | RSA数字签名 | 审批人无法否认自己的操作 |
| **高可用性** | 分布式架构 | 部分节点故障系统仍可运行 |

### 区块链 vs 传统系统安全对比

| 安全维度 | 传统集中式系统 | 本区块链系统 |
|---------|--------------|-------------|
| 数据存储 | 单点存储，风险集中 | 多节点分布，风险分散 |
| 篡改检测 | 难以发现 | 哈希验证自动检测 |
| 审计追溯 | 日志可被删除 | 区块链记录不可删除 |
| 故障恢复 | 单点故障即停��� | 节点自动切换 |
| 内部威胁 | DBA可直接修改数据 | 需多节点共识 |
| 责任认定 | 难以举证 | 数字签名不可抵赖 |

## 快速开始

```bash
# 1. 克隆项目
git clone https://github.com/your-repo/blockchain-exam-system.git

# 2. 进入项目目录
cd blockchain-exam-system/blockchain-exam-node1

# 3. 配置数据库和Redis (修改 application.yml)

# 4. 初始化数据库
mysql -u root -p < docs/database.sql

# 5. 运行项目
mvn clean install
mvn spring-boot:run

# 6. 访问API文档
# http://localhost:8080/api/doc.html
```
